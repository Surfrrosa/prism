Prism Privacy Policy

Last updated: February 26, 2026

The short version

Prism makes zero network requests. Your data never leaves your browser. We don't collect, transmit, or sell anything.

What data is stored

When you visit a recognized news website, Prism stores a reading record locally in your browser's IndexedDB. Each record contains:

  • The page URL and title
  • The domain name
  • A timestamp
  • The source's bias rating and credibility score (from our bundled database)
  • The source name

Your settings (time period preference and notification toggle) are stored in chrome.storage.local.

What data is NOT collected

  • No browsing data is transmitted to any server
  • No analytics or telemetry
  • No cookies or tracking pixels
  • No personal information (name, email, accounts)
  • No page content is read or stored
  • No third-party services are contacted

Network requests

Prism makes exactly zero network requests. No fetch, XHR, or WebSocket calls. No external fonts, scripts, or resources. Everything is bundled locally in the extension.

Permissions

  • tabs - Read the URL when you navigate to a page, to match against our news source database. This is the core functionality.
  • storage - Store your settings locally.
  • alarms - Schedule weekly report notifications and daily data cleanup.
  • notifications - Deliver your optional weekly media diet report.
  • sidePanel - Display the full dashboard.

Data retention

Reading records older than 90 days are automatically pruned. You can clear all data at any time from the dashboard. Uninstalling the extension removes all stored data.

Source database

The media bias database is bundled as static data at build time from the Media Bias/Fact Check open-source project (MIT license). It is not fetched from any remote server. Database updates require a new extension version.

Open source

Prism is open source. You can inspect the full source code at github.com/Surfrrosa/prism.

Changes

If this policy changes, the update date at the top will be revised. Material changes will be noted in the extension's changelog.
